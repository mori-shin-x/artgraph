# examples/speckit-integration — Spec Kit + artgraph

End-to-end walkthrough of `artgraph integrate speckit`: how the integration is
installed, what gets added to `.specify/extensions.yml`, when each hook fires
during the Spec Kit workflow, and how to gate `/speckit-implement` on drift.

## The three hooks

`artgraph integrate speckit` registers two hooks by default and a third
opt-in gate:

| Spec Kit checkpoint | Hook                | What artgraph does                            |
| ------------------- | ------------------- | --------------------------------------------- |
| `after_tasks`       | `scan && reconcile` | Snapshots a fresh `.trace.lock` baseline      |
| `after_implement`   | `check --diff`      | Reports drift / orphans / uncovered on the diff |
| `before_implement`  | `check --gate`      | Blocks `/speckit-implement` on drift (opt-in via `--gate`) |

## Layout

```
examples/speckit-integration/
├── .artgraph.json
└── .specify/
    ├── extensions.yml             # starting state — empty installed list
    └── specs/001-auth/
        ├── spec.md                # FR-001 / FR-002
        └── tasks.md               # T001 / T002 with `@impl(FR-…)`
```

`.specify/extensions/artgraph/` is generated by `integrate speckit` and is
intentionally **not** committed — it's listed in `examples/.gitignore` so the
example always starts from a clean state.

## Step 1 — Install the integration

```bash
cd examples/speckit-integration
node ../../dist/cli.js integrate speckit
```

> In your own project this is just `npx artgraph integrate speckit`. The
> `node ../../dist/cli.js` form is so the example runs against the local build.

Expected output:

```
✓ Integrated: speckit (Spec Kit)

Created (5):
  .specify/extensions/artgraph/extension.yml
  .specify/extensions/artgraph/README.md
  .specify/extensions/artgraph/commands/artgraph.scan-reconcile.md
  .specify/extensions/artgraph/commands/artgraph.check-diff.md
  .specify/extensions/artgraph/commands/artgraph.check-gate.md

Modified (1):
  .specify/extensions.yml

Next:
  Run /speckit-tasks to verify the after_tasks hook fires
  Run /speckit-implement to verify the after_implement hook fires
```

### What changed in `extensions.yml`

Before:

```yaml
installed: []
settings:
  auto_execute_hooks: true
hooks: {}
```

After `integrate speckit`:

```yaml
installed:
- artgraph
settings:
  auto_execute_hooks: true
hooks:
  after_tasks:
  - extension: artgraph
    command: artgraph.scan-reconcile
    enabled: true
    optional: false
    priority: 50
    prompt: Run artgraph scan && reconcile to refresh trace baseline?
    description: Refresh artgraph baseline after tasks
    condition: null
  after_implement:
  - extension: artgraph
    command: artgraph.check-diff
    enabled: true
    optional: false
    priority: 50
    prompt: Run artgraph check --diff to verify coverage/orphan/drift?
    description: Verify artgraph traceability after implementation
    condition: null
```

Other extensions' hooks in `extensions.yml` are never touched. The integration
is **idempotent** — re-running `integrate speckit` reports
`Already integrated: speckit (Spec Kit) — no changes` and leaves the disk
byte-for-byte identical.

## Step 2 — Walk the Spec Kit workflow

```bash
# Spec Kit fires `after_tasks` after /speckit-tasks runs:
node ../../dist/cli.js scan && node ../../dist/cli.js reconcile
```

```
Nodes: 6  Edges: 6
  req: 2  doc: 2  file: 0  test: 0  task: 2
Lock file updated: .trace.lock
```

Two `req` nodes (FR-001, FR-002), two `doc` nodes (spec / tasks), two `task`
nodes (T001, T002) parsed from `tasks.md` with their `@impl(FR-…)` tags
captured as edges. No source / test files yet.

```bash
# /speckit-implement runs the agent, then fires `after_implement`:
node ../../dist/cli.js check --diff
```

With nothing implemented yet, both requirements are flagged:

```
UNCOVERED:
  FR-001
  FR-002
COVERAGE:
  FR-001: untagged
  FR-002: untagged
```

After the agent adds `src/auth.ts` with `// @impl FR-001` and a test with
`it("[FR-001] …")`, `check` narrows to just the remaining work:

```
UNCOVERED:
  FR-002
COVERAGE:
  FR-001: verified
  FR-002: untagged
```

## Step 3 — Add the `--gate`

To **block** `/speckit-implement` whenever drift / orphans / uncovered are
present (instead of just reporting after the fact):

```bash
node ../../dist/cli.js integrate speckit --gate    # add before_implement gate
node ../../dist/cli.js integrate speckit --no-gate # remove it again
```

`--gate` is _declarative_: setting it adds the hook, `--no-gate` removes it,
omitting the flag leaves whatever state is already on disk. The added entry:

```yaml
  before_implement:
  - extension: artgraph
    command: artgraph.check-gate
    enabled: true
    optional: false
    priority: 50
    prompt: "Gate: run artgraph check --gate before implementing?"
    description: Gate implementation on artgraph traceability
    condition: null
```

`check --gate` exits with code 2 when drift / uncovered are present, so Spec
Kit halts before `/speckit-implement` runs.

## Step 4 — Uninstall

```bash
node ../../dist/cli.js integrate speckit --uninstall
```

Removes `.specify/extensions/artgraph/` and every `artgraph` entry from
`.specify/extensions.yml`. Hooks from other extensions are preserved.
